# 中国汽车销量分析平台：数据资产审计与扩容报告

## 1. 当前数据资产现状 (Current Status)

目前项目已打通以下核心数据流：
- **基础销量流**: 已采集 2024-2026 年（含预测/模拟）的月度销量数据（Gasgoo/CPCA 归口）。
- **实时行情流**: 已集成 `yahoo-finance2`，实现 Top 车企实时市值与年度营收的自动化同步。
- **关联关系链**: 建立了“车企-车型-销量”的级联索引，支撑市场集中度与产品生命周期计算。

---

## 2. 核心数据缺失项分析 (Identified Gaps)

经过对行业分析报告标准的对比，本项目在以下四个维度存在显著数据缺失：

### A. 财务深度数据 (Financial Depth)
- **缺失项**: 单车毛利率、研发支出占比、库存周转率、自由现金流。
- **影响**: 当前估值仅停留于 PS (营收) 维度，无法进行 PE (市盈率) 或更深维度的经营效率（Alpha 收益）对标。

### B. 产品/配置元属性 (Product Metadata)
- **缺失项**: 动力能量形式 (BEV/PHEV/ICE)、车型定位 (SUV/Sedan/MPV)、电池容量、智驾等级。
- **影响**: 无法进行“细分赛道”的渗透率分析（如：20万以上纯电 SUV 的市场占有率演变）。

### C. 真实终端价格 (Terminal Pricing)
- **缺失项**: 区域终端实际成交价、限时促销权益 (如：2万置换补贴)、厂家金融政策。
- **影响**: `/pricing` 页面目前依赖博弈模型驱动。缺失真实数据将导致无法量化“价格战”与“销量弹性”的精准函数关系。

### D. 行业宏观因子 (Macro Drivers)
- **缺失项**: 油电差价指数、充电桩覆盖密度趋势、出口海运费指数、各地以旧换新政策阈值。
- **影响**: 缺乏宏观变量对銷量波动的解释性回归分析。

## 3. 数据获取计划 (Acquisition Roadmap)

针对上述缺失项，我们将采取“三位一体”的采集策略：

### 阶段一：金融数据自动化 (API 驱动 - 即刻实施)
- **来源**: 雅虎财经 `quoteSummary` 与 `financialData` 模块。
- **目标**: 自动补全各车企的季度财报指标。
- **技术**: 扩展现有 `/api/sales/valuation` 逻辑。

### 阶段二：产品配置与终端价监控 (网爬/第三方 - 中期任务)
- **来源**: 汽车之家 (Autohome)、懂车帝 (Dongchedi)、易车网 (Yiche)。
- **技术**: 
    - 使用 `Firecrawl` 或 `Sub-agent` 扫描特定车型的“落地价”论坛数据。
    - 抓取“配置表”生成标准化的车型标签库。
- **入库**: 填充 `PriceRecord` 表。

### 阶段三：宏观政策与上险量对齐 (官方/半官方 - 长期协同)
- **来源**: 乘联会 (CPCA) 深度月报、保险行业协会 (CBIRC) 上险量数据。
- **技术**: 
    - 开发 PDF 解析器定时处理乘联会公开的 PPT/PDF 报告。
    - 接入 CnEVPost 等行业聚合媒体的 API。
